Module 1: The Model and the Harness.
Before I explain anything — no pressure, just a guess — when you hear the words "model" and "harness," does either one mean anything to you yet? Even "no idea" is a perfectly good answer; it just tells me where to begin.
Here's the core idea, in plain terms:
Underneath everything is a large language model — a Claude model. On its own, a model does *exactly one thing*: you give it some text, and it predicts what text comes next. That's the whole trick. It can't open a file. It can't run a command. It can't remember yesterday. It's a very good guesser about *language*, and nothing else.
So how does that turn into a thing that edits your code, runs your tests, and picks up where you left off this morning?
That's the harness. Claude Code is the harness wrapped *around* the model. The harness is everything that isn't the model itself: it feeds text to the model, reads what the model says, and — here's the key part — when the model says "I'd like to read the file data/trials.tsv," the harness is what actually goes and opens that file, then hands the contents back to the model as more text.
One picture that helps: the model is the engine; the harness is the car. The wheels, the steering, the dashboard — that's the harness. You don't drive an engine; you drive a car. And almost everything you'll learn in this whole tour is part of the *harness*, not the model.

Now, Module 2: The System Prompt — and it follows directly from what you just nailed.
Here's the puzzle it solves. You said it perfectly: the model on its own is just a text-predictor. So here's the thing — it doesn't even *know* it's "Claude Code." It doesn't know it has a harness, or tools, or that it's allowed to edit files, or that it should work in back-and-forth turns with a person. A raw model has none of that built in.
So *something* has to tell it all that — before you ever type a single word. That something is the system prompt.
The system prompt is a block of instructions the harness slips in at the very front of the text the model sees, every session. It's where the model gets told:
- its role — "you're an agent that helps with software engineering tasks,"
- what tools exist and how to ask for them,
- the environment — what operating system, what folder you're in, today's date,
- how to behave — be concise, confirm before risky actions, follow the project's rules.
That bundle of instructions is a big part of what turns "a model that predicts text" into "an agent that gets things done." The agent-ness isn't baked into the model's weights — it's largely *instructions*, handed to it fresh at the start of every session.
And here's the one thing I most want you to walk away with: you don't write the system prompt. Anthropic writes and maintains it, and it changes as Claude Code gets updated. You generally never see it and never edit it. Knowing it *exists* is useful; knowing it's *not your lever* is the actual lesson — because later in the tour you'll meet the parts that *are* yours to write (that's CLAUDE.md, coming up).
So, the picture so far is a little stack of text the harness assembles for the model:
```
SYSTEM PROMPT (Anthropic's) ← role, tools, environment, behavior
CLAUDE.md (yours) ← coming in Module 4
the conversation so far ← grows as you talk
```
