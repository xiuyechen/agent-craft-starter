From zero to the tutor.
Five steps. A checkpoint after each.

This gets you from nothing installed to the moment the tutor starts quizzing you — on a Mac or a Windows machine, even if you've never opened a terminal. It doesn't teach the material; that's the homework's job. This is just the plumbing.

~20–30 min, first time Mac & Windows No coding required

One thing to know first

Claude Code needs a paid Claude plan — Pro or Max (or a Team / Enterprise / Console account). The free Claude.ai plan won't work: if you only have a free account, sign-in in Step 3 gets rejected, and the fix is the plan, not your computer.

If your lab or course gave you a specific login or key, use that — ask whoever ran the workshop. You don't have to sort this out this second; just know a free account is the one thing that won't go through.

01/install the editor

Install VS Code

VS Code is the window you'll run everything inside. It's free. Go to code.visualstudio.com and click Download — it detects your system.

Mac
Open the downloaded .zip, then drag Visual Studio Code into your Applications folder. Open it from there (right-click → Open the first time if macOS warns you).
Win
Run the .exe (the "User Installer") and accept the defaults. If it offers "Add to PATH," tick it.
You'll know it worked when VS Code opens and you see the Welcome tab.
02/install claude code

Install Claude Code

First open a terminal — you'll paste one line into it.

Mac
Open the Terminal app: press ⌘ Space, type "Terminal," hit Enter.
Win
Open PowerShell: Start menu, type "PowerShell," hit Enter. Your prompt starts with PS C:\ — that's how you know it's PowerShell.

Paste the line for your system and press Enter. This is the official installer; it sets up a self-contained Claude Code — you do not need to install Node.js or anything else first.

curl -fsSL https://claude.ai/install.sh | bash # Mac
PS> irm https://claude.ai/install.ps1 | iex # Windows PowerShell
You'll know it worked when Close the terminal and open a new one (the installer adds claude to your PATH, which only takes effect in fresh terminals), then run claude --version — a version number means you're set.
Still "command not found" in a fresh terminal? The installer couldn't wire up your PATH (common on Mac). Run these two lines once — they tell your shell where claude lives — then try claude --version again:
$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc $ source ~/.zshrc
(That's for the Mac default shell, zsh. On Linux/bash, use ~/.bashrc in both lines.)
Prefer no terminal at all? There's a Desktop app (Mac/Windows) with a normal graphical interface — install that, sign in, open this repo's folder in it. The steps are the same; "run /teach-me" just happens in its chat box. On Mac with Homebrew? brew install --cask claude-code also works.
03/sign in

Sign in to Claude

Just run:

claude

The first time, it opens a browser and walks you through signing in with your Claude account (or pasting an API key, if that's what your lab gave you). It remembers you after that.

You'll know it worked when You land at the Claude Code prompt — a > waiting for input — with no auth error. Type /help to confirm it responds, then /exit to leave. You'll come back in inside the repo in Step 5.
Sign-in rejected? This is the one step that's about your account, not your machine. A free Claude account won't work — you need Pro/Max, or the login your lab gave you. Reinstalling won't help here.
04/get the repo

Download this repo and open it

"Cloning" just means downloading the repo as a folder you can edit. You need git — check first:

git --version

A version number means you have it. If not:

Mac
Running git --version makes macOS offer to install the developer command-line tools — accept it.
Win
Install Git for Windows from git-scm.com/download/win (defaults are fine), then open a fresh PowerShell.

Now clone it. Your home folder is a fine place:

$ cd ~ $ git clone https://github.com/xiuyechen/agent-craft-starter.git

Then open it in VS Code: File → Open Folder… and select agent-craft-starter.

You'll know it worked when VS Code's left sidebar (the Explorer) lists the repo's files — README.md, HOMEWORK.md, curriculum/, data/, and so on.
05/run the tutor

Open a terminal inside the repo — then run the tutor

This is the step beginners get wrong most often. Claude Code only knows about this project if its terminal is opened inside this folder. The safe way: use VS Code's built-in terminal, because it opens in the folder you have open. Menu Terminal → New Terminal (or the shortcut Ctrl + ` — that's the backtick, top-left of the keyboard).

Look at the path the terminal shows. It must end in agent-craft-starter. If it doesn't, run cd ~/agent-craft-starter to fix it. Then start Claude Code:

claude

First, prove the context loaded — ask it, in plain English:

> what does this project do, and where does its data live?
You'll know it worked when It answers with specifics — a posterior parietal cortex decision task, data in data/, an open question in notes.md. It knows that only because the repo's CLAUDE.md pointed it there. A vague "I'd need to look at the files" means your terminal isn't inside the repo — check the path again.

Now run the tutor:

/teach-me
Nothing happens? Type it in plain English instead: teach me the curriculum in this repo.
Done — this is the finish line The tutor greets you, offers the two tracks, and starts asking you questions. That's setup complete. Now go do the homework.

When something breaks

// one honest try, then bring the exact error to the workshop

claude: command not found
First, close every terminal and open a new one. Still broken on Mac? Your PATH wasn't set — run echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc then source ~/.zshrc (see Step 2).
Sign-in is rejected
Account-side, not your machine. A free Claude account won't work — you need Pro/Max, or the login your lab gave you.
Claude gives generic answers about the project
Your terminal isn't inside the repo. Use VS Code's built-in terminal and confirm the path ends in agent-craft-starter.
/teach-me does nothing
Type "teach me the curriculum in this repo" in plain English instead.
git: command not found
Install git (Step 4) — macOS offers it automatically; Windows gets it from git-scm.com.
You're set up. The learning starts now.

Every checkpoint you just hit was the same move: check the world, don't assume the command worked. That's Principle 04 — you were already doing Agent Craft before the tutor said a word.
